# EasyCon / ezcon 接入 TODO

## 目标

在 `auto_bdsp_rng` 中提供一个内置的“伊机控模块”，让用户可以在本软件里完成伊机控脚本的选择、编辑、参数填写、运行、停止、日志查看和连接状态确认。

用户不需要直接打开 EasyCon GUI，也不需要手动使用命令行。第一阶段底层可以调用 `ezcon.exe run` 完成脚本执行；后续如果全自动流程被 CLI 的单次连接限制卡住，再切换为常驻连接的 Bridge / Server 后端。

## 产品原则

- [ ] 用户视角中只有“连接伊机控”和“运行脚本”，不暴露 `ezcon.exe run`、临时脚本路径等底层细节。
- [ ] UI 体验参考 EasyCon 1.61 GUI 的核心区域：脚本编辑区、输出日志区、运行按钮、串口连接区、底部状态区。
- [ ] 第一版允许底层只是 CLI 调用，但 UI 和后端接口必须按未来 Bridge 后端可替换的方式设计。
- [ ] 原始脚本默认作为模板保护，不在运行时直接覆盖。
- [ ] 运行前必须保存最终脚本内容到文件，因为 CLI 只能读取脚本文件执行。
- [ ] 运行日志必须留在本软件中，用户不需要回到命令行窗口看输出。
- [ ] 当前项目第一阶段不接入识图、OCR、ImgLabel、摄像头、VS Code 插件或 EasyCon GUI。

## 当前已确认事实

- [x] 本机 `EASYCON_ROOT` 可以指向 EasyCon 安装目录，例如 `D:\app\伊机控\1.61`。
- [x] `ezcon.exe` 位于 `%EASYCON_ROOT%\ezcon.exe`。
- [x] `ezcon` 不一定在 `PATH` 中，因此软件不能只依赖 `ezcon` 命令名。
- [x] `ezcon --version` 可以输出版本，例如 `1.6.1+f744ae4...`。
- [x] `ezcon run <script> -p COMx` 每次启动一个进程、连接一次单片机、执行一个脚本，然后退出。
- [x] CLI 模式不能在一次串口连接中连续执行多个脚本。
- [x] 当前 `script` 目录中的脚本不需要识图，不依赖 `@xxx` 搜图变量或 `ImgLabel` 目录。
- [x] 现有脚本的动态参数主要位于脚本顶部，例如 `_闪帧`、`_翻几次`、`_过几次`、`_等待时间`。

## 用户流程规划

### 首次打开软件

- [ ] 软件启动后自动检测 EasyCon：
  - [ ] 优先读取用户保存的 `ezcon.exe` 绝对路径。
  - [ ] 其次读取 `EASYCON_ROOT\ezcon.exe`。
  - [ ] 最后尝试从 `PATH` 查找 `ezcon`。
- [ ] 检测成功后显示 EasyCon 版本。
- [ ] 检测失败时显示明确错误和“选择 ezcon.exe”入口。
- [ ] 自动刷新串口列表。
- [ ] 自动选择上次使用的串口；如果不存在，则提示用户选择。
- [ ] 用户可以点击“连接伊机控”或“准备连接”。

### 普通脚本运行

- [ ] 用户从内置脚本列表选择脚本，或将脚本拖拽到伊机控模块。
- [ ] 软件加载脚本内容到中央编辑区。
- [ ] 软件扫描脚本顶部参数，自动生成参数输入区。
- [ ] 用户填写或修改参数。
- [ ] 用户可以直接编辑脚本文本。
- [ ] 点击“运行脚本”时：
  - [ ] 将当前编辑区内容保存为临时 `.ecs` 文件。
  - [ ] 如参数输入区有修改，先同步回脚本文本。
  - [ ] 调用 CLI 后端运行临时脚本。
  - [ ] 输出区显示实时日志。
  - [ ] 运行按钮切换为停止按钮。
  - [ ] 计时器开始计时。
- [ ] 脚本完成后：
  - [ ] 显示完成状态。
  - [ ] 停止计时。
  - [ ] 保留日志。
  - [ ] 允许用户再次运行或修改参数后复跑。

### RNG 联动运行

- [ ] 用户在 RNG 结果中选中目标。
- [ ] 软件根据当前 Seed、目标 advances、offset、场景类型计算脚本参数。
- [ ] 自动选择合适脚本或提示用户选择脚本。
- [ ] 自动填入 `_闪帧`、`_翻几次`、`_过几次`、`_等待时间` 等参数。
- [ ] 运行前展示最终参数，允许用户手动微调。
- [ ] 点击运行后与普通脚本运行流程一致。

### CLI 后端下的连接体验

- [ ] UI 中可以显示“准备就绪”，但要在内部区分“真实常驻连接”和“CLI 待运行”。
- [ ] CLI 后端下，每次执行脚本都会重新连接单片机。
- [ ] 如果用户的单片机需要每次连接前按 reset，第一阶段无法完全免除该动作。
- [ ] UI 文案要避免误导，不应声称 CLI 后端已经长期保持串口连接。

### Bridge 后端下的连接体验

- [ ] 后续 Bridge 后端实现后，用户点击“连接伊机控”即真正打开并保持串口连接。
- [ ] 手动按键、脚本执行、连续任务都复用同一个连接。
- [ ] UI 不需要大改，只切换后端实现。

## 后端架构规划

### 建议模块划分

```text
auto_bdsp_rng/
  automation/
    easycon/
      __init__.py
      backend.py          # 后端抽象接口和状态模型
```

### CLI 后端限制

- [ ] `ezcon run` 不能保持串口连接。
- [ ] 每运行一个脚本都会启动一个新进程并重新连接单片机。
- [ ] 如果单片机每次连接前都需要手动 reset，CLI 后端无法彻底消除该问题。
- [ ] 第一阶段不适合拆分成多个短脚本连续执行。
- [ ] 第一阶段应尽量将一次自动化任务合并成一个完整 ECS 脚本。

## 脚本模板与参数规划

### 现有脚本目录

- [ ] 默认扫描 `D:\codex_project\auto_bdsp_rng\script`。
- [ ] UI 中展示内置脚本列表。
- [ ] 后续打包时要确认 `script` 目录被包含进发布包。
- [ ] 允许用户打开外部脚本，但外部脚本不进入内置脚本列表。

### 参数识别规则

- [ ] 扫描脚本中的常量赋值行：

```text
_闪帧 = 填入这里
_翻几次 = 填入这里
_过几次 = 填入这里
_等待时间 = 8
```

- [ ] 识别格式：

```text
^\s*(?P<name>_[^=\s]+)\s*=\s*(?P<value>.*?)(?P<comment>\s+#.*)?$
```

- [ ] 将 `填入这里` 视为必填参数。
- [ ] 将整数默认值视为数字参数。
- [ ] 将其他默认值视为文本参数。
- [ ] 保留行尾注释并作为 UI 帮助文本。
- [ ] 暂不处理复杂表达式赋值。
- [ ] 遇到无法识别的内容时不报错，只是不生成对应参数控件。

### 参数写回规则

- [ ] 参数改动后，只替换赋值右侧，不改变量名和注释。
- [ ] 示例：

```text
_闪帧 = 填入这里              # 在这里输入你距离闪帧还差多少
```

替换为：

```text
_闪帧 = 123              # 在这里输入你距离闪帧还差多少
```

- [ ] 写回时保留原行缩进。
- [ ] 写回时保留原行尾注释。
- [ ] 写回后编辑区内容同步更新。

### 临时脚本生成

- [ ] 不直接修改 `script` 目录中的原始脚本。
- [ ] 运行前保存最终内容到临时 `.ecs` 文件。
- [ ] 第一阶段生成目录可使用：

```text
script\.generated
```

- [ ] 后续打包版可改为用户缓存目录，例如：

```text
%LOCALAPPDATA%\auto_bdsp_rng\easycon\generated
```

- [ ] 生成文件名包含：
  - [ ] 原脚本名。
  - [ ] 时间戳。
  - [ ] 可选任务类型。
- [ ] 保持 UTF-8 编码。
- [ ] 尽量保留原脚本换行风格。
- [ ] 生成后可选调用 `ezcon.exe format` 验证语法。
- [ ] 运行失败时保留临时脚本，便于调试。
- [ ] 运行成功后可按配置保留最近 N 个临时脚本。

## 错误处理规划

- [ ] 找不到 `ezcon.exe`：
  - [ ] 显示“请选择 ezcon.exe 或设置 EASYCON_ROOT”。
  - [ ] 禁用运行按钮。
- [ ] `ezcon --version` 失败：
  - [ ] 显示路径可能无效或文件损坏。
- [ ] 串口为空：
  - [ ] 禁用运行按钮，除非启用 mock 模式。
- [ ] `ezcon run` 返回非 0：
  - [ ] 标记任务失败。
  - [ ] 保留 stdout/stderr。
  - [ ] 保留临时脚本。
- [ ] CLI 输出“单片机连接失败”：
  - [ ] 状态显示连接失败。

- [ ] 脚本编译失败：

- [ ] 用户停止：
  - [ ] 终止子进程。
  - [ ] 标记为已中止，而不是失败。

## 第二阶段：Bridge / Server 后端

当需要真正全自动连续执行多个脚本时，需要实现常驻连接后端。

### 目标

- [ ] 启动一个常驻 EasyCon Bridge 进程。
- [ ] Bridge 只连接一次指定 COM 口。
- [ ] 主程序通过本地 IPC 向 Bridge 发送脚本文本或手柄动作。
- [ ] Bridge 连续执行多个任务，不释放串口。
- [ ] UI 持续显示真实连接状态、当前动作和日志。

### 可选实现方案

- [ ] fork `ezcon`，新增 `server` 模式。
- [ ] 新建独立 `.NET` 项目，引用 EasyCon 核心库实现 `EasyConBridge.exe`。
- [ ] 使用 HTTP localhost、WebSocket、Named Pipe 或 stdin/stdout JSON 作为 IPC。

### Bridge 最小接口

```text
version()
list_ports()
connect(port)
disconnect()
status()
run_script(script_text)
press(button, duration_ms)
stick(side, direction_or_angle, duration_ms)
stop()
logs()
```

### Bridge UI 增强

- [ ] 显示真实“已连接/断开/执行中”状态。
- [ ] 支持手动按钮实时发送，不再为每个按钮启动一次 CLI。
- [ ] 支持脚本队列。
- [ ] 支持连续任务自动执行。
- [ ] 支持紧急停止当前任务。

### Bridge 启动条件

- [ ] CLI 第一阶段已经完成。
- [ ] 已确认单次连接限制阻碍目标全自动流程。
- [ ] 已确认 EasyCon 核心库可以被稳定引用或 fork。
- [ ] 已确定 IPC 协议。

## 推荐实施顺序

1. 完成 CLI 后端的路径检测、版本检测、串口选择。
2. 完成脚本模板参数解析和临时脚本生成。
3. 完成 UI 中的脚本拖拽、参数输入、脚本编辑、运行、日志显示。
