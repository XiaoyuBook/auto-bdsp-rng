## 目标

在 `auto_bdsp_rng` 中提供一个内置的“伊机控模块”，让用户可以在本软件里完成伊机控脚本的选择、编辑、参数填写、运行、停止、日志查看和连接状态确认。

用户不需要直接打开 EasyCon GUI，也不需要手动使用命令行。最终交付目标必须支持“打开软件后连接一次单片机，后续多次运行脚本都复用同一条连接”。`ezcon.exe run` 只能作为过渡验证和兼容后端，不能作为最终全自动后端。

- [x] 用户视角中只有“连接伊机控”和“运行脚本”，不暴露 `ezcon.exe run`、临时脚本路径等底层细节。（2026-05-05，耗时 1 分钟，Bridge 主路径已隐藏 CLI 细节；CLI 诊断模式仍保留）
- [ ] UI 体验参考 EasyCon 1.61 GUI 的核心区域：脚本编辑区、输出日志区、运行按钮、串口连接区、底部状态区。
- [x] CLI 调用只作为过渡验证和兼容后端，最终必须实现常驻连接后端。（2026-05-05，耗时 1 分钟，UI 已切换为 Bridge 优先）
- [x] UI 和后端接口必须按 Bridge / Server 后端为主设计，而不是围绕 CLI 单次执行设计。（2026-05-05，耗时 1 分钟）
