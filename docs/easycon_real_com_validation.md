# EasyCon 真实 COM 验证清单

> 目的：记录真实串口验证流程。没有真实单片机和用户确认前，只能完成流程准备，不能把 CLI 验证写成最终长期连接验收。

## 真实 COM 最小脚本验证流程

- [ ] 确认 `ezcon.exe` 路径有效，版本可读取。
- [ ] 刷新串口列表，记录目标串口，例如 `COM7`。
- [ ] 关闭可能占用串口的 EasyCon GUI、VS Code 插件或其他串口工具。
- [ ] 如当前单片机固件要求，先按一次 reset。
- [ ] 生成最小脚本：

```text
WAIT 50
```

- [ ] 使用 CLI 过渡后端运行：`ezcon.exe run <minimal.ecs> -p COMx`。
- [ ] 记录 stdout、stderr、exit code、开始时间、结束时间、耗时、脚本路径和串口。
- [ ] 若失败，按错误类型记录：串口占用、未 reset、连接超时、脚本编译失败或其他。
- [ ] 明确结论：此项只证明真实 COM 单次 CLI 运行可用，不证明长期连接。
